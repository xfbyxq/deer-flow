本模块是 `app.gateway.auth` 包内的共享层，被本地/ OIDC/GitHub 等具体 Provider 和路由层共同依赖，自身不暴露 HTTP 端点。
- 配置：`config.py` 通过 Pydantic `AuthConfig` 模型 + 全局 `_auth_config` 单例加载 `AUTH_JWT_SECRET`，若未设置则从 `{base_dir}/.jwt_secret`（0600 权限）持久化生成；`get_auth_config()` 懒初始化，`set_auth_config()` 供测试注入。
- 抽象边界：`providers.py` 定义 `AuthProvider` ABC（`authenticate` / `get_user`），通过运行时 import `User` 避免循环引用，作为所有具体 Provider 的统一契约。
- 错误体系：`errors.py` 集中声明 `AuthErrorCode`、`TokenError` 两个 StrEnum 及 `AuthErrorResponse` Pydantic 模型，并维护 `token_error_to_code` 映射，作为整个认证子系统的单一错误来源。
- 安全写盘：`credential_file.py` 封装原子写入 `admin_initial_credentials.txt`（0600）的工具函数，避免将明文密码输出到日志聚合系统。
- 运维工具：`reset_admin.py` 是可独立运行的 CLI（`python -m app.gateway.auth.reset_admin`），直接连接 `deerflow.persistence.engine` 更新 `UserRow` 并调用 `write_initial_credentials(label="reset")` 落盘新密码。
- OIDC 预配：`user_provisioning.py` 实现 `get_or_provision_oidc_user`，按顺序执行「已绑定检查 → 邮箱验证要求 → 域名白名单 → 本地账号冲突(409) → 自动创建」，并通过 `LocalAuthProvider` 访问存储，不直接依赖 SQLAlchemy。
依赖方向：本模块仅向上依赖 `deerflow.config.*`、`deerflow.persistence.*`、`fastapi`、`pydantic`，不被其他业务模块反向依赖，属于自底向上的共享库。
