模块由三个职责清晰的子文件组成：
- `jwt.py`：网关内部访问令牌（HS256）的创建与解码，依赖 `app.gateway.auth.config.get_auth_config()` 读取 `jwt_secret` 与过期时间，返回 Pydantic `TokenPayload` 或 `TokenError`。
- `oidc_state.py`：无状态 OIDC 中间态管理，将 state / nonce / PKCE code_verifier / next_path 打包为 `OIDCStatePayload`，用同一份 `jwt_secret` 以 HS256 签名后写入 HttpOnly Cookie（前缀 `df_oidc_state_`），通过 `is_secure_request` 判断是否设置 secure 标志，从而避免 Redis 等外部存储。
- `oidc.py`：对外暴露的 `OIDCService` 服务类，封装完整的 OIDC 授权码流程——发现（`.well-known/openid-configuration`）、构建授权 URL、交换 access/id token、从 JWKS 解析公钥并校验 ID Token（强制 issuer 回指、白名单算法 RS*/ES*、nonce 常量时间比较）、可选拉取 UserInfo 并校验 sub 一致性；对 provider metadata 与 JWKS 使用内存 dict + TTL 缓存，HTTP 客户端为 `httpx.AsyncClient(timeout=15s)`。
依赖方向：`oidc_state.py` → `config` + `csrf_middleware`；`jwt.py` → `config` + `errors`；`oidc.py` 仅依赖标准库、`httpx`、`PyJWT`，不反向依赖另外两个文件，三者共同被上层 gateway handler 组合调用。
