- 所有共享单例通过 `app.state.<attr>` + `deps.get_<attr>()` 访问器暴露，缺失时返回 503，禁止直接 `getattr(request.app.state, ...)` 散落各处
- 路由文件只负责声明端点，业务逻辑委托给 `services.py` 或 `deerflow.*` 层，保持 router 薄且可测试
- 需要管理员权限的路由统一调用 `deps.require_admin_user`，不再在各路由内重复判断 `user.system_role == 'admin'`
- 可选功能路由（如 GitHub webhook）通过 `is_route_enabled()` 守卫 `include_router`，未配置时 URL 返回 404 而非 403，遵循 fail-closed 原则
