四个文件各自定义一个 `APIRouter(prefix="/api", tags=[...])`，由上层网关聚合挂载，职责边界清晰：
- `agents.py`：通过 `_require_agents_api_enabled()` 统一受 `agents_api.enabled` 配置门控，读写用户隔离的 `config.yaml` + `SOUL.md`（使用 `asyncio.to_thread` 把阻塞 IO 移出事件循环），并维护全局 `USER.md`；名称经正则校验后归一化为小写。
- `skills.py`：以 `SkillStorage` 抽象区分 PUBLIC（全局 `extensions_config.json`）与 CUSTOM/LEGACY（每用户 `_skill_states.json`）两类持久化；所有写操作先走 `scan_skill_content` 安全扫描，再写入并通过 `refresh_user_skills_system_prompt_cache_async` 刷新缓存；管理员接口通过 `app.gateway.deps.require_admin_user` 守卫。
- `console.py`：纯只读报表层，依赖 SQLAlchemy 查询 `runs` / `threads_meta` 表，未启用 SQL 后端时返回 503；内置基于 `models[*].pricing` 的模型级成本估算（支持 prompt-cache-hit 折扣），按本地时区窗口聚合 token 用量。
- `features.py`：仅暴露 `/api/features` 供前端在启动时探测哪些可选功能可用，避免无谓请求被 403 拒绝。
依赖方向单向向外：路由层 → `deerflow.config` / `deerflow.persistence` / `deerflow.skills.*` / `app.gateway.authz`，不反向依赖业务执行路径。
