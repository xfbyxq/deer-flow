- `app.py::create_app` 是唯一的装配点：按固定顺序注册 `AuthMiddleware → CSRFMiddleware → CORSMiddleware → TraceMiddleware`，再 `include_router` 挂载各子模块路由；GitHub webhook 路由仅在配置了 secret 时挂载，作为 fail-closed 的安全网。
- `deps.py::langgraph_runtime` 通过 `AsyncExitStack` 集中创建并挂到 `app.state` 的 LangGraph 基础设施（StreamBridge、Checkpointer、Store、RunManager、RunEventStore、ThreadMetaStore、各类 Repository），并在 shutdown 阶段先 drain 进行中的 run 再关闭数据库连接，避免池关闭竞态。
- 请求期依赖全部走 `get_*` 访问器（如 `get_run_manager`、`get_thread_store`、`get_config`），缺失时返回 503；`AppConfig` 不缓存于 `app.state`，而是每次请求经 `get_app_config()` 热重载，但引擎等重启才生效的单例绑定启动时的 `startup_config` 快照，形成明确的“热重载边界”。
- 认证子系统 (`auth/`) 通过 `get_local_provider()` 懒加载本地提供者，`deps.get_current_user_from_request` 从 cookie 解码 JWT 并校验 token_version，供 `auth_middleware` 与各路由复用。
- GitHub Webhook 集成 (`github/`) 由 `routers/github_webhooks.py` 在 include 时触发 import side-effect 注册 ChannelRunPolicy，随后由 `lifespan` 启动的 ChannelService 消费事件，实现 webhook → agent fan-out 的跨模块编排。
