本模块是 Next.js App Router 下的 React Client Components 集合，按功能划分为若干子目录：
- 布局层：`workspace-container.tsx` 导出 `WorkspaceContainer`/`WorkspaceHeader`/`WorkspaceBody`，配合 `workspace-sidebar.tsx` 通过 `@/components/ui/sidebar` 的 `Sidebar` 组合出完整 Shell；`workspace-header.tsx` 在侧边栏折叠态下显示 DF Logo 并渲染“新建对话”入口。
- 业务原语：`input-box.tsx` 是最复杂的组件，基于 `@/components/ai-elements/prompt-input` 封装，集成模型选择、技能斜杠命令、follow-up 建议、上传限制校验、goal 命令（PUT/DELETE GET `/api/threads/:id/goal`）以及 sidecar 引用上下文注入；`code-editor.tsx` 基于 `@uiw/react-codemirror` + `@codemirror/*` 语言扩展，根据 next-themes 动态切换 Monokai/BareLight 主题，并在流式加载时回退到只读 `Textarea`。
- 领域子包：`agents/`、`artifacts/`、`changes/`、`channels/`、`chats/`、`citations/`、`messages/`、`settings/`、`sidecar/` 各自以 `index.ts` 聚合导出，形成按领域划分的内聚子模块。
- 依赖方向：仅向上依赖 `@/core/*`（i18n、models、skills、uploads、threads、sidecar、config）和 `@/components/ui/*`（shadcn），不反向引用其他 workspace 子包，保持扁平可组合。
- 状态来源：通过 `./messages/context` 的 `useThread` 获取当前线程消息与 loading 状态，作为多数组件共享的运行时上下文。
