本模块是 Next.js App Router 的 workspace 子应用壳层，按职责拆为三文件：
- `layout.tsx`：服务端异步布局，调用 `getServerSideUser()` 做统一鉴权分流（authenticated → 注入 `AuthProvider` + `WorkspaceContent`；needs_setup/system_setup_required/unauthenticated → redirect；gateway_unavailable → 使用 `GatewayOfflineFallback` 包裹并透传 `gatewayUnavailable` 给内容层；config_error → 抛错），并通过 `dynamic = 'force-dynamic'` 禁用缓存。
- `page.tsx`：根路由仅做重定向——在 `NEXT_PUBLIC_STATIC_WEBSITE_ONLY=true` 时扫描 `public/demo/threads` 首个目录跳转到 `/workspace/chats/<dir>`，否则一律跳到 `/workspace/chats/new`。
- `workspace-content.tsx`：客户端组件，读取 `sidebar_state` cookie 初始化 `SidebarProvider`，组合 `WorkspaceSidebar` + `SidebarInset` 作为主区域，并在 inset 顶部挂载 `GatewayOfflineBanner`，同时提供 `CommandPalette` 和 `Toaster`。
依赖方向单向向下：layout → workspace-content → 各 `@/components/workspace/*` 子组件，不反向引用。
