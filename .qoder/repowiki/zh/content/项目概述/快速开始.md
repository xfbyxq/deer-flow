## 简介
DeerFlow 是一个开源的“超级智能体编排框架”，通过子智能体、持久化记忆、沙箱执行和可扩展技能，帮助用户完成从研究到内容生成、自动化工作流等广泛任务。本指南面向新用户，帮助你在最短时间内完成环境准备、交互式配置、并成功启动第一个 DeerFlow 实例（支持 Docker 与本地两种路径）。

## 环境要求
- Python 3.12+
- Node.js 22+
- pnpm（前端依赖管理）
- uv（Python 包管理与运行）
- Docker（推荐用于隔离执行与生产部署）
- Nginx（由脚本统一代理，通常随服务脚本或容器使用）

说明：
- 后端与前端分别有独立的依赖与构建流程；推荐使用仓库顶层 Makefile 命令进行一键式检查与安装。
- 若仅使用 Docker 模式，可跳过本地 Node/uv 的安装，但需要确保 Docker 守护进程可用。

## 安装与初始化
以下命令在仓库根目录执行：

- 拉取代码并进入项目根目录
- 运行交互式配置向导，生成最小可用的 config.yaml 并写入 .env 中的密钥占位
- 可选：运行健康检查与诊断工具

常用命令
- make setup：启动交互式配置向导（LLM 提供商、搜索、执行模式、IM 通道等）
- make doctor：验证当前环境并给出修复建议
- make support-bundle：收集诊断信息，便于提交问题

注意
- 向导会提示是否覆盖已有配置；如已存在 config.yaml，可选择保留或重新配置。
- 向导完成后，会输出下一步建议（例如安装依赖、启动服务等）。

## 交互式配置向导
向导分为若干步骤，逐步引导你完成关键配置：

- LLM 提供商选择与模型配置
  - 支持多种提供商与模型列表；部分提供商需填写 base_url 或额外能力开关（如思考/推理）。
  - 向导会询问是否需要启用“思考/推理”能力，并在 config.yaml 中写入相应标记。
- Web 搜索与网页抓取（可选）
  - 可选择一个或多个搜索/抓取提供商；若未配置，不影响基础对话能力。
  - 向导会将所需环境变量写入 .env。
- 执行与安全设置
  - 选择执行模式：本地沙箱（更快，直接访问宿主机文件系统）或容器沙箱（更隔离，需要 Docker）。
  - 决定是否启用 bash 命令执行与文件写入工具（write_file、str_replace）。
- IM 通道连接（可选）
  - 可在浏览器侧边栏启用 Telegram、Slack、Discord、飞书/Lark、钉钉、微信、企业微信等通道；凭据可在后续通过 UI 配置。

输出产物
- 根目录生成 config.yaml（包含模型、工具组、沙箱、技能、标题生成、摘要、子智能体、记忆等核心配置项）
- 根目录生成 .env（包含模型与工具的 API Key 占位）
- 前端 .env 示例文件（如不存在则自动创建）

## 运行方式：Docker 部署
推荐用于生产与稳定运行，具备热重载的开发镜像与完整的前后端、Nginx 反向代理。

- 开发模式（源码挂载、热重载）
  - 首次拉取沙箱镜像：make docker-init
  - 启动服务：make docker-start
  - 访问地址：http://localhost:2026
- 生产模式（本地构建镜像，挂载运行时配置与数据）
  - 构建并启动所有服务：make up
  - 停止服务：make down
  - 访问地址：http://localhost:2026

重要说明
- 默认单 Gateway Worker；如需多 Worker，请结合 Redis Stream Bridge 并注意跨 Worker 限制（取消、去重、IM 状态等仍为 Worker 局部）。
- 数据库后端建议使用 sqlite 或 postgres；LangGraph Checkpointer/Store 与应用数据共享同一后端。
- 如需跨源访问，可通过 GATEWAY_CORS_ORIGINS 指定允许的 Origin。

## 运行方式：本地开发
适合开发者快速迭代，前后端与 Nginx 均由脚本统一管理。

前置条件
- 完成“安装与初始化”步骤，确保存在有效的 config.yaml。
- Windows 用户请使用 Git Bash 执行脚本（原生 cmd/PowerShell 不受支持）。

常用命令
- make check：检查 Node.js、pnpm、uv、nginx 等依赖
- make install：安装后端与前端依赖，并安装预提交钩子
- make dev：启动 Gateway + Frontend + Nginx（默认端口 2026）
- make doctor：验证环境并获取修复建议

访问地址
- http://localhost:2026

## 常见问题与故障排除
- Docker 权限问题（Linux）
  - 现象：连接 Docker 守护进程时出现 permission denied
  - 处理：将当前用户加入 docker 组并重新登录，再重试
- 缺少系统依赖
  - 现象：make check 报告缺失 node、pnpm、uv、nginx 等
  - 处理：按提示安装对应工具后重试
- 无模型配置
  - 现象：config.yaml 未配置 models 或未提供 API Key
  - 处理：通过向导或手动编辑 config.yaml，并在 .env 中填入对应变量名
- 跨源访问失败
  - 现象：浏览器跨域报错
  - 处理：设置 GATEWAY_CORS_ORIGINS 为逗号分隔的精确 Origin
- 多 Worker 行为差异
  - 说明：Redis Stream Bridge 解决 SSE 投递与 Last-Event-ID 回放，但取消、去重、IM 状态仍为 Worker 局部；建议单 Worker 或显式粘性路由后再提升 GATEWAY_WORKERS
